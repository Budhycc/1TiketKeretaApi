Aplikasi: 1TiketKeretaApi (Sistem Pemesanan Tiket Kereta Api)

JAR yang Dapat Dieksekusi: dist/1TiketKeretaApi.jar

Layar Awal:
- Layar login dengan kolom "Username" dan "Password".
- Autentikasi dilakukan terhadap tabel `admin` di database.

Elemen Navigasi Utama (setelah login, di jendela `Home`):
- Formulir pemesanan tiket dengan berbagai kolom input.
- Bilah menu:
  - Menu "File":
    - "Admin": Mengarah ke bagian admin.
    - "Keluar": Keluar dari aplikasi.
  - Menu "Data Penumpang": Mengarah ke bagian melihat data penumpang.

Fitur Inti yang Tampak:
1.  Login: Mengautentikasi pengguna (kemungkinan administrator).
2.  Pemesanan Tiket:
    - Detail penumpang (ID, nama, alamat).
    - Pemilihan stasiun (awal dan tujuan).
    - Pemilihan kereta dan gerbong.
    - Pemilihan tanggal.
    - Jumlah penumpang dewasa/anak-anak.
3.  Perhitungan Harga: Secara dinamis menghitung harga tiket berdasarkan opsi yang dipilih (kemungkinan dari tabel `harga` di database).
4.  Pembuatan Tiket: Menyimpan pemesanan ke tabel `penumpang` dan menghasilkan laporan tiket (`Tiket.jasper`).
5.  Panel Admin (`Admin.java`): Diakses melalui menu "File" -> "Admin".
    - Manajemen Pengguna Admin:
        - Menambah, melihat, mengedit, dan menghapus pengguna admin (data disimpan di tabel `admin`).
        - Antarmuka Pengguna (UI): Kolom input untuk nama, username, password; JTable untuk tampilan; Tombol Simpan, Edit, Hapus.
    - Manajemen Harga:
        - Melihat dan mengedit harga tiket (data disimpan di tabel `harga`).
        - Harga dikategorikan berdasarkan "Status" (misalnya, dewasa/anak) dan "Kelas" (kelas gerbong).
        - UI: JTable untuk tampilan; kolom untuk status, kelas, harga (harga dapat diedit); Tombol Simpan.
6.  Melihat Data Penumpang (`Penumpang.java`): Diakses melalui menu "Data Penumpang".
    - Melihat semua catatan pemesanan penumpang dari tabel `penumpang` di JTable.
    - Kolom meliputi ID, nama, alamat, kereta, gerbong, tujuan, waktu berangkat/tiba, tanggal, jumlah dewasa/anak, dan total harga.
    - Mencetak Laporan Penumpang: Menghasilkan laporan JasperReport (`penumpang.jasper`) dari semua data penumpang.
    - Menghapus Semua Data Penumpang: Menghapus tabel `penumpang` setelah konfirmasi.

Catatan Teknis:
- Aplikasi ini adalah aplikasi GUI Java Swing.
- Menggunakan database MySQL (membutuhkan driver `mysql-connector-java`).
- Detail koneksi database ditangani di `tiket.Koneksi`.
- Logika aplikasi utama setelah login ada di `tiket.Home`.
- Menggunakan JasperReports untuk pembuatan tiket dan daftar penumpang.
- Menjalankan JAR secara langsung di lingkungan headless (tanpa antarmuka grafis) menyebabkan `java.awt.HeadlessException`. Eksplorasi dilakukan dengan menganalisis kode sumber.
- Aplikasi tampaknya memiliki stasiun awal tetap: "Bandung (BD)" dalam formulir pemesanan.
- Opsi kereta default: "Argo Wilis", "Argo Parahyangan", "Lodaya".
- Opsi gerbong default: "Eksekutif 1/2", "Ekonomi 1/2/3".
- Harga untuk dewasa dan anak-anak diambil berdasarkan kelas gerbong dari tabel `harga`.
- `JTextField bayangan1` di `Admin.java` digunakan sebagai kolom tersembunyi untuk menyimpan ID item yang saat ini dipilih di tabel untuk operasi pembaruan/penghapusan.
- Navigasi antar bagian utama (Home, Admin, Penumpang) dilakukan dengan membuat instance JFrame baru dan membuang yang sekarang.
- Bagian `Admin` dan `Penumpang` juga memiliki opsi menu untuk menavigasi ke bagian lain atau keluar.
